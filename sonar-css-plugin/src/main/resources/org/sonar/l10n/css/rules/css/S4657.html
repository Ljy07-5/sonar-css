在普通属性（longhand properties）之后定义的简写属性（shorthand properties）将完全覆盖普通属性（longhand properties）中定义的值， 使得一个普通属性（longhand property）无效。应该重构代码以考虑普通属性（longhand properties），或者完全删除它。

不合规的代码示例

a {
  padding-left: 10px;
  padding: 20px; /* 不合规的；padding 会覆盖 padding-left，使其无效 */
}

合规的解决方案

a {
  padding: 10px; /* 合规的；Compliant; padding 定义了一般填充运行状况，然后 padding-left，在这之后，精确地定义了左边的填充情况 */
  padding-left: 20px;
}
