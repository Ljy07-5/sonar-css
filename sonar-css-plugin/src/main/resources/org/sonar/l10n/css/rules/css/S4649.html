如果用户的浏览器上没有 font 或 font-family 声明中已定义的字体名称，则浏览器将其使用默认字体显示文本。 建议始终为每个 font 或者 font-family 的声明定义一个通用字体系列名称，来得到比依赖浏览器默认字体更少的 字体降级情况。 所有浏览器应该支持一个通用字体列表，即匹配以下字体族： Serif, Sans-serif, cursive, fantasy, Monospace.

不合规的代码示例

a {
  font-family: Helvetica, Arial, Verdana, Tahoma; /* 不合规的；列表中没有通用字体系列名称 */
}

合规的解决方案

a {
  font-family: Helvetica, Arial, Verdana, Tahoma, sans-serif;
}
